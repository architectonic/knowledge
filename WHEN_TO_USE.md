# When to Use Knowledge

## Use ordinary knowledge when

- updates happen through deliberate curation;
- source changes are occasional;
- a maintainer can review changes during normal work;
- the main need is to preserve claims, sources, evidence, contradiction, and uncertainty.

Examples:

- project architecture and decisions;
- a stable internal glossary;
- bounded research notes;
- a personal or organizational second brain;
- source-backed wiki articles that do not require recurring source watching.

## Add skills and meta when

The corpus needs repeatable acquisition, compilation, query, evaluation, audit, migration, or retirement procedures.

```text
knowledge-system = knowledge + skills + meta
```

## Add living knowledge when

Correctness decays on an external clock: regulations, markets, security advisories, provider catalogs, or other changing sources.

## Graphs and retrieval

Canonical files should remain understandable without a database. Derived systems may provide:

```text
lexical retrieval
vector retrieval
graph traversal
community summaries
Obsidian visualization
Graphify or GraphRAG projections
```

Evaluate those paths against representative questions. Relevance, similarity, and generated summaries are not evidence by themselves.
